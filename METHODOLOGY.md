# Metodología: Vibe Coding & Operational Integrity

Nuestra forma de construir software es un equilibrio entre la agilidad creativa y el rigor institucional.

## 🌊 Vibe Coding: El "Flow" del Desarrollo
El *Vibe Coding* no es falta de estructura, es **flexibilidad inteligente**.

1.  **Iteración Rápida**: Construimos prototipos funcionales mínimos (MVP) para validar la lógica operativa de inmediato.
2.  **Intuición Técnica**: Aprovechamos la experiencia en seguridad física para "sentir" las vulnerabilidades antes de que aparezcan en el código.
3.  **Refactorización Continua**: Una vez que la "vibe" del sistema es sólida, aplicamos los estándares de seguridad más estrictos.

## 🛡️ Operational Integrity: El Blindaje
Cada proyecto debe seguir estos pasos de integridad:

### Fase 1: El Modelo de Amenazas (Threat Model)
Antes de escribir el core, identificamos quién querría romper el sistema. Para **Ojo de Paz**, por ejemplo, el riesgo es la suplantación de identidad para realizar ataques aéreos erróneos.

### Fase 2: Implementación de la Capa de Confianza
- **Hardware First**: Si es posible, la identidad debe nacer en el hardware.
- **Criptografía Robusta**: Usar solo algoritmos estándar (AES, SHA, ECDSA). Nunca inventar criptografía propia.

### Fase 3: Auditoría de Auditoría
Implementamos logs de auditoría persistentes que no solo registren éxitos, sino cada intento de validación fallido, permitiendo un análisis forense posterior.

## 🏗 Cómo Aplicamos esto a Cada Commit
- **Claridad**: Un commit debe ser una unidad funcional lógica.
- **Autoridad**: El código debe demostrar dominio técnico.
- **Propósito**: Cada cambio debe acercarnos a la meta de impacto social definida en el proyecto.
